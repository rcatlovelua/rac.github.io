Rcat's Anti-Cheat (RAC)

Server-side anti-cheat system for Roblox Studio with a focus on security, modularity, and exploit resistance.

Roblox Lua Server-Side Anti-Cheat Obfuscation

Overview

Rcat's Anti-Cheat (RAC) is designed to detect and mitigate common exploit techniques used in Roblox games. The system is primarily server-based, ensuring that critical validation and decisions are never trusted to the client.

Architecture

Server Script

The server script is the core of RAC and runs inside ServerScriptService.

  • Validates movement and physics
  • Detects speed, fly, noclip, and teleport exploits
  • Handles punishments (kick, ban, warnings)
  • Manages excluded players

Client Script

The client script provides limited information to the server. All client data is treated as untrusted and verified server-side.

  • State reporting
  • Anti-tamper signaling
  • No authority over detections

Excluded Players

RAC supports an exclude list that allows specific players to bypass anti-cheat checks. 

ExcludedPlayers = {
    "PlayerName1",
    "PlayerName2"
}

Configuration

return {
    Enabled = true,
    Debug = true,

    SpeedLimit = 30,
    FlyCheck = true,
    TeleportCheck = true,

    KickOnDetect = true,
    BanOnRepeat = false
}

Obfuscation

Client-side scripts should always be obfuscated. Exploiters can decompile LocalScripts and inspect logic. 

script.Source = [[
    -- LOL! SCRIPT IS OBFUSCATED
    print("Nice try")
]]

Obfuscation is strongly recommended for production environments.

Disclaimer

No anti-cheat system can guarantee complete protection. RAC should always be combined with proper server-side validation.
