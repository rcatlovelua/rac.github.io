Rcat's Anti-Cheat (RAC)

Server-side anti-cheat system for Roblox Studio, focused on security, performance, and clean modular architecture.

Roblox Lua Server-Side Anti-Cheat

Overview

Rcat's Anti-Cheat (RAC) is designed to detect and mitigate common exploit techniques used in Roblox games. The system is primarily server-based, ensuring that critical validation and decisions are never trusted to the client.

Architecture

Server Script

The server script is the core of RAC. It runs inside ServerScriptService and is responsible for:

  • Validating player movement and behavior
  • Detecting speed, fly, noclip, and teleport exploits
  • Handling punishment logic (kick, ban, warnings)
  • Managing excluded players

Client Script

The client script exists only to provide limited data to the server and should never contain trusted logic. All client data is verified server-side.

  • Basic state reporting
  • Anti-tamper signals
  • No authority over detection decisions

Excluded Players

RAC supports an exclude list that allows specific players to bypass anti-cheat checks. This is useful for developers, testers, or administrators. 

ExcludedPlayers = {
    "PlayerName1",
    "PlayerName2"
}

Excluded players will not trigger detections or punishments.

Configuration

All major settings are stored in a centralized configuration module. 

return {
    Enabled = true,
    Debug = true,

    SpeedLimit = 30,
    FlyCheck = true,
    TeleportCheck = true,

    KickOnDetect = true,
    BanOnRepeat = false
}
